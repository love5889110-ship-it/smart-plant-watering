## 一、产品概述

### 1.1 产品定位

智能浇花系统是一款面向家庭/阳台植物爱好者的 **个性化自动养护平台**，通过 IoT 传感器实时感知土壤状态，结合多因子决策引擎，为每株植物提供量身定制的浇水建议与自动执行能力，并以拟人化角色方式呈现植物状态，提升养护趣味性。

### 1.2 核心价值

| 价值 | 说明 |
|------|------|
| 不再忘浇水 | 传感器实时监控，缺水自动提醒/浇水 |
| 不再乱浇水 | 多因子决策，结合植物习性、季节、时段个性化决策 |
| 过程透明可见 | 每次浇水的决策因子完整记录，养护历史可追溯 |
| 养花有趣味 | 植物拟人化角色，有情绪、会说话，随时间/季节变化 |

### 1.3 目标用户

- 家庭/阳台植物爱好者，养 1-4 株盆栽
- 经常外出、容易忘记浇水的上班族
- 希望精细化养护、了解植物状态的爱好者

## 二、系统架构

```
ESP32 硬件层
  ├── 电容式土壤湿度传感器 (GPIO34)
  ├── 水泵控制 (GPIO25 + 继电器)
  ├── WiFi + MQTT 上报数据
  └── BLE 蓝牙直连备用通道

后端服务层 (FastAPI + SQLite)
  ├── MQTT Handler：接收传感器数据
  ├── 多因子决策引擎：计算浇水建议
  ├── REST API：供前端调用
  └── 植物知识库：植物特性/季节参数

前端展示层 (Vue 3 + Tailwind)
  ├── Dashboard：总览 + 今日任务
  └── PlantDetail：详情 + 决策面板 + 养护记录
```

## 三、功能需求

### 3.1 核心功能（已实现）

#### F1. 多因子智能浇水决策

系统综合 6 个因子加权评分，决定是否浇水及浇水时长：

| 因子 | 权重 | 内容 |
|------|------|------|
| 湿度缺口 | 45% | 当前湿度与目标区间的偏差 |
| 季节系数 | 15% | 春夏蒸发旺需水多，冬季休眠需水少 |
| 浇水时段 | 10% | 清晨/上午最佳，正午/夜间扣分 |
| 植物策略 | 15% | 保湿型 vs 干透浇透型 |
| 浇水间隔 | 10% | 防止过频，紧急时可突破 |
| 健康趋势 | 5% | 健康分低时增加关注 |

- 综合分 ≥ 0.15 建议浇水
- block 级因子（过湿/间隔未到）直接否决
- 紧急缺水可突破间隔限制

#### F2. 植物个性化档案

支持 3 种植物类型（可扩展）：

| 植物 | 策略 | 目标湿度 | 特点 |
|------|------|---------|------|
| 蓝莓 🫐 | 保湿 | 40-65% | 干旱敏感，果期需水量大 |
| 映山红 🌸 | 干透浇透 | 35-55% | 极怕积水，根系易腐烂 |
| 普通花卉 🌿 | 标准 | 35-60% | 通用模板 |

每种植物有独立的：季节需水系数、基础浇水时长、最短浇水间隔、干旱/积水敏感度

#### F3. 实时状态监控

- 传感器 30 秒上报一次湿度
- 状态分 5 级：critical_dry / dry / ok / wet / critical_wet
- 健康评分（0-100）基于近 7 天湿度在目标区间内的比例
- 养成等级（Lv.1-5）随健康分提升

#### F4. 植物角色化呈现

- 每株植物有专属 emoji 表情和对话气泡
- 情绪感知时间段（清晨/上午/正午/下午/傍晚/夜晚）
- 情绪感知季节（春/夏/秋/冬）
- 主页动态背景随时间+季节变化
- 季节漂浮粒子动画（樱花/阳光/落叶/雪花）

#### F5. 养护记录可视化

- 完整浇水历史记录
- 每次浇水保存决策因子快照
- 历史记录可展开查看"当时为何浇水"
- 近 7 天湿度趋势折线图

#### F6. 手动控制

- 手动浇水（3/5/10/15/30 秒可选）
- 自动浇水开关（按多因子评分触发）
- 快速浇水按钮（从建议卡片直接触发，8 秒）

## 四、硬件方案

### 4.1 当前方案（单植物）

```
ESP32 Dev Board
├── GPIO34 ← 电容式土壤湿度传感器
├── GPIO25 → 继电器 → 5V 微型水泵
├── GPIO2  → LED 状态指示
└── 5V USB 供电
```

### 4.2 多植物扩展方案

**方案 A：重力滴灌 + 电磁阀（推荐）**
```
高处水桶（重力）
  ↓
分水器
  ├── 5V电磁阀1 → 植物A（GPIO25）
  ├── 5V电磁阀2 → 植物B（GPIO26）
  └── 5V电磁阀3 → 植物C（GPIO27）

传感器：
  ├── 湿度传感器A → GPIO34
  ├── 湿度传感器B → GPIO35
  └── 湿度传感器C → GPIO32
```
- 零水泵，重力驱动，无噪音
- 5V 电磁阀可 ESP32 直驱，无需继电器
- 新增每株成本约 ¥25

**方案 B：多 ESP32 独立部署**
- 每株植物独立 ESP32，WiFi 上报不同 device_id
- 后端已支持，软件零改动
- 适合植物距离较远的场景

## 六、MQTT 通信协议

| Topic | 方向 | 说明 |
|-------|------|------|
| `plant/{device_id}/sensor` | ESP32→后端 | 湿度上报 `{moisture_pct, raw, rssi}` |
| `plant/{device_id}/pump/cmd` | 后端→ESP32 | 浇水指令 `{duration_ms, reason}` |
| `plant/{device_id}/pump/status` | ESP32→后端 | 泵状态反馈 |
| `plant/{device_id}/heartbeat` | ESP32→后端 | 心跳/在线状态 |

## 八、启动指南

### 快速启动（开发环境）

```bash
# 1. MQTT Broker
cd backend && docker-compose up -d

# 2. 后端
cd backend && source venv/bin/activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 3. 前端
cd frontend && npm run dev
# 浏览器打开 http://localhost:5173
```

### ESP32 烧录

1. 修改 `firmware/src/config.h`：填入 WiFi 和 MQTT Broker IP
2. PlatformIO 烧录固件
3. 串口监视器做传感器校准（干燥/湿润各记录 raw 值）
4. 重新烧录，完成
